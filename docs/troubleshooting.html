Troubleshooting

Fixes for the most common hiccups.

Opening the app on macOS

The first time you open Python Chess on macOS, you'll probably see a warning saying the app "cannot be opened because it is from an unidentified developer." That's just because the app isn't signed with an Apple developer certificate, it doesn't mean anything's actually wrong. Here's how to get past it, two ways:

Option A: System Settings

  1. Double-click PythonChess.app. Click OK to dismiss the warning.
  2. Open System Settings → Privacy & Security and scroll down to the Security section. You'll see a message saying the app was blocked, click Open Anyway.
  3. One last confirmation pops up. Click Open, and the app launches. macOS remembers this choice after that, so you only need to do it once.

Option B: Terminal

Open Terminal and run:

xattr -cr PythonChess.app

Then double-click the app like normal. This just strips the quarantine flag macOS slaps on anything downloaded from the internet.

Analysis or Stockfish isn't available

Live analysis and the Stockfish opponent both need the Stockfish engine set up first. If you haven't done that yet, the app will show a one-time message explaining it rather than just failing quietly. Open Preferences and use the built-in downloader to grab the right version for your computer automatically, no manual installation needed.

Finding your files

Python Chess keeps your saved games, exported PGN files, and a diagnostic log in a per-user folder on your computer. You shouldn't normally need to go digging around in here, but it's useful if something's gone wrong or you want to back up your games.

Exported PGN files

SystemLocation
Linux~/.local/share/python-chess/pgn/
macOS~/Library/Application Support/python-chess/pgn/
Windows%APPDATA%\python-chess\pgn\

Each export gets a timestamp in its name (something like python-chess_20260722_143000.pgn), so exporting the same game twice keeps both copies instead of overwriting one.

Diagnostic log

If something's acting up, especially if the app closes unexpectedly, this is the first place to check:

SystemLocation
Linux~/.local/state/python-chess/log/python-chess.log
macOS~/Library/Logs/python-chess/python-chess.log
Windows%LOCALAPPDATA%\python-chess\Logs\python-chess.log
Reporting a problem

Run into a bug? Please open an issue on GitHub. Pasting in anything relevant from the log file above helps a lot.
